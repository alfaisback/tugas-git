RANGKUMAN GIT

1. Git config

digunakan untuk mengatur konfigurasi tertentu sesuai keinginan pengguna, seperti email, algoritma untuk diff, username, format file,

2. Git init

digunakan untuk membuat repository di file lokal.

3. git status

digunakan untuk mengetahui sebuah status dari sebuah repository

4. Git add

adalah perintah yang digunakan untuk menambahkan file baru di repository yang dipilih,

5. Git clone

digunakan untuk menunjuk ke repo yang ada dan membuat tiruan atau salinan repo itu di direktori baru, di lokasi lain. Repositori asli dapat ditemukan di sistem file lokal atau pada protokol pendukung yang dapat diakses mesin jarak jauh. Perintah git clone menyalin repositori Git yang ada.

6. Git commit

commit adalah unit blok penyusun inti dari garis waktu proyek Git. commit dapat dianggap sebagai snapshot atau tonggak sepanjang garis waktu proyek Git. commit dibuat dengan perintah git commit untuk menangkap status proyek pada saat itu.

7. Git push

digunakan untuk mengunggah konten repositori lokal ke repositori jarak jauh. Mendorong adalah cara Anda mentransfer komit dari repositori lokal Anda ke repo jarak jauh. Ini adalah pasangan dari git fetch , tetapi saat mengambil impor dilakukan ke cabang lokal, mendorong ekspor dilakukan ke cabang jarak jauh.

8. Git config

di gunakan untuk menavigasi di antara cabang yang dibuat oleh git branch . Memeriksa cabang memperbarui file di direktori kerja agar sesuai dengan versi yang disimpan di cabang itu, dan memberi tahu Git untuk merekam semua komitmen baru di cabang itu.
